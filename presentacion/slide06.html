Decisiones Técnicas

Por qué se eligió cada tecnología — Comparativas y beneficios

⚡FastAPI Elegido

Framework async con validación automática vía Pydantic. Genera Swagger/OpenAPI sin código extra. Rendimiento comparable a Go/Node.

FastAPI Flask Django Tornado
Async nativo Pydantic v2 Auto-docs Type hints
🧠Ollama (LLM primario) Elegido

LLM 100% local. Sin costos por token, privacidad total de datos BBVA. Swap a OpenAI/Groq con una variable gracias a Strategy Pattern.

Ollama (local) OpenAI API Groq API HuggingFace
Sin costo Privacidad Offline Swap fácil
📦ChromaDB Elegido

Vector store embebido, cero infraestructura. Persistencia en disco, API Python nativa. Ideal para prototipos y pruebas técnicas.

ChromaDB Pinecone Weaviate FAISS Qdrant
Embebido Gratis Persistencia Metadata filter
🕷️Selenium + BS4 Elegido

BBVA.com.co usa JavaScript dinámico. Scrapy no renderiza JS. Selenium headless + BS4 para parsing garantiza contenido completo.

Selenium Scrapy Playwright requests
JS rendering Headless Docker ready Dynamic content
🗄️SQLite + WAL Elegido

Zero-config, embebido, perfecto para prueba técnica. WAL mode permite lecturas concurrentes. Migrar a PostgreSQL = nuevo adapter.

SQLite PostgreSQL MongoDB Redis
Zero config WAL mode Portátil ACID
🏛️Arquitectura Hexagonal Elegido

Máximo desacoplamiento. El dominio no conoce la infra. Permite testear sin DB/LLM reales y cambiar cualquier componente sin refactoring.

Hexagonal Layered (MVC) Clean Arch Monolítico
DIP nativo Testable Extensible SOLID
📐Sentence-Transformers Elegido

Embeddings multilingüe de alta calidad. Modelo local sin costos por API. Cross-Encoder para reranking fino (Top-K → Top 3).

Sentence-Transformers OpenAI Embeddings Cohere
Local Multilingüe Reranking Sin costo
🐳Docker Compose Elegido

Un solo docker compose up levanta todo. Multi-stage builds reducen imagen 60%. Healthchecks aseguran disponibilidad.

Docker Compose Manual install Kubernetes Podman
Reproducible Multi-stage Healthchecks 1 comando
